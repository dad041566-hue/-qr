# QR테이블 서비스 — Figma AI + HTML/CSS/JS 개발 인계서

## 1. 프로젝트 목적

QR테이블 서비스의 랜딩페이지 및 서비스 소개 페이지를 제작한다. 이 페이지는 단순 홍보용이 아니라 실제 영업, 제안, 도입 문의 전환을 목표로 한다. 사용자는 외식업 대표, 카페 운영자, 주점 운영자, 프랜차이즈 본사 담당자이며, 핵심 메시지는 `QR 주문 + 호출 + 주방 전달 + 매출 연동 + 정산 관리`를 한 번에 해결하는 통합 솔루션이라는 점이다.

---

## 2. 페이지 목표

* 대표가 봤을 때 도입 효과가 명확하게 보이도록 설계
* 고객이 봤을 때 서비스 흐름을 쉽게 이해하도록 설계
* 문의 전환율을 높이는 구조로 설계
* 개발 시 HTML / CSS / JS로 바로 구현 가능한 구조로 설계

## 4. Figma AI용 메인 프롬프트

아래 문장을 그대로 Figma AI에 넣어서 시작한다.

```text
Design a modern responsive landing page for a Korean B2B SaaS service called 'QR테이블 통합 운영 시스템'. The service provides QR table ordering, staff call, kitchen order transmission, table management, sales integration, and settlement dashboard for restaurants, cafes, pubs, and franchise stores.

Create a premium, clean, conversion-focused website with a Korean business SaaS style. Use a modern layout similar to high-end Korean startup websites. The page should feel trustworthy, practical, and scalable.

Visual style:
- clean white background with subtle gray sections
- strong primary navy or deep blue accent
- secondary soft sky blue or mint highlights
- rounded cards
- soft shadows
- clear section spacing
- premium enterprise SaaS feeling
- minimal but visually rich
- easy-to-read Korean typography

Page sections:
1. Header with logo, navigation, CTA button
2. Hero section with headline, subheadline, CTA buttons, dashboard mockup preview
3. Core value cards showing main benefits
4. Service flow section explaining QR scan → menu order → kitchen → payment → settlement
5. Feature grid section with 6 to 8 features
6. Benefit comparison section for customer and store owner
7. Admin dashboard preview section with KPI cards, order status, table status, sales chart
8. Use case section for restaurant, cafe, pub, franchise
9. CTA section for consultation and demo request
10. FAQ section
11. Footer with company info and quick links

Components:
- responsive header
- large hero typography
- dashboard cards
- feature cards with icons
- comparison blocks
- process timeline
- mock mobile screen cards
- charts and admin tables
- strong CTA buttons
- FAQ accordion

Design responsive layouts for desktop, tablet, and mobile.
Use auto layout, reusable components, clear layer naming, 8px spacing system, and consistent design tokens.
```

## 7. 섹션별 상세 구성

## 7-1. Header

구성 요소:

* 좌측 로고
* 중앙 메뉴: 서비스 소개 / 주요 기능 / 도입 효과 / 활용 업종 / 문의하기
* 우측 버튼: 데모 요청 / 상담 문의

개발 구조:

* HTML: `header.site-header > .container > .header-inner`
* CSS: sticky header, blur background, bottom border
* JS: scroll 시 shadow 추가, 모바일 메뉴 toggle

## 7-2. Hero Section

구성 요소:

* 메인 카피
* 서브카피
* CTA 버튼 2개
* 우측 대시보드 목업
* 하단 신뢰 배지 또는 핵심 수치

추천 카피:

* QR주문부터 매출연동·정산까지 한 번에
* 주문, 호출, 주방 전달, 결제, 운영 데이터를 하나의 흐름으로 연결하세요.

개발 구조:

* HTML: `section.hero`
* 내부: `hero-copy`, `hero-actions`, `hero-visual`
* JS: CTA 버튼 hover, 숫자 카운팅 애니메이션 가능

## 13. 개발자 인계용 체크포인트

* 모든 섹션은 container width 1200~1280px 기준
* spacing system 8px 기준으로 통일
* 버튼 variant 명확화
* hover / active / focus 상태 정의
* FAQ는 하나만 열리도록 또는 다중 열림 선택 가능
* 모바일에서 CTA는 최소 44px 터치 높이 확보
* 텍스트 대비 WCAG 고려
* 카드 내부 여백 통일
* 섹션 사이 여백 리듬 유지

## 16. 최종 방향

이 페이지는 단순 회사소개가 아니라 `매장 대표가 도입 문의를 남기게 만드는 영업형 랜딩페이지`로 설계해야 한다. 따라서 디자인은 예쁜 것보다 신뢰감, 기능 명확성, KPI 시각화, CTA 전환 구조에 집중한다.
