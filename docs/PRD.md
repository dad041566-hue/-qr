# PRD: 테이블QR (TableFlow)

**문서 버전:** 2.0
**작성일:** 2026-03-15
**상태:** 확정

---

## 1. 개요

**TableFlow** — 외식업 자영업자를 위한 QR 주문 + POS + 대기 관리 서비스.

### 운영 모델

Self-service SaaS가 아닌 **개발사 관리형 서비스**:
- 개발사가 슈퍼어드민 패널에서 매장 계정 생성 및 이용 기간 직접 관리
- 점주는 개발사로부터 계정(이메일+임시비번)을 받아 로그인
- 결제·구독 기능 없음 — 계약은 오프라인, 시스템은 기간 만료 여부로만 접근 제어

---

## 2. 사용자

| 역할 | 설명 | 접근 |
|------|------|------|
| **슈퍼어드민** | 개발사 운영팀 | `/superadmin` — 매장·계정·기간 전체 관리 |
| **점주 (owner)** | 매장 오너 | `/admin` — 대시보드 전체 권한 |
| **매니저 (manager)** | 홀/주방 매니저 | `/admin` — 주문·메뉴 관리, 매출 조회 |
| **직원 (staff)** | 홀·주방 스태프 | `/admin` — 주문 관리만 |
| **고객** | QR 스캔 방문객 | `/m/:storeSlug/:tableId` — 주문만 |

### 역할별 권한

| 역할 | 주문 관리 | 메뉴 수정 | 직원 관리 | 매출 조회 |
|------|----------|----------|----------|----------|
| owner | ✅ | ✅ | ✅ | ✅ |
| manager | ✅ | ✅ | ❌ | ✅ |
| staff | ✅ | ❌ | ❌ | ❌ |

## 3. 계정 플로우

### 점주 계정 생성 (개발사 처리)
```
슈퍼어드민 → 매장 생성 → 점주 계정 생성 (이메일 + 임시비번)
→ 이용 기간 설정 (시작일 / 만료일)
→ 점주에게 계정 정보 전달 (오프라인)
→ 점주 첫 로그인 → 비밀번호 강제 변경
```

### 직원 계정 생성 (점주 처리)
```
점주 어드민 → 직원 계정 생성 (이메일 + 임시비번)
→ 직원에게 계정 정보 전달
→ 직원 첫 로그인 → 비밀번호 강제 변경
```

### 이용 기간 만료
```
만료일 초과 → 로그인 차단 (슈퍼어드민 제외)
→ 고객 QR 주문 페이지도 비활성화
→ 슈퍼어드민이 기간 연장 시 즉시 복구
```

## 4. 기능 요구사항

### 4-1. 슈퍼어드민 패널 (`/superadmin`) — 개발사 전용

| 기능 | 설명 |
|------|------|
| 매장 목록 | 전체 매장 현황 (이름, 슬러그, 기간, 상태) |
| 매장 생성/수정 | 매장명, slug, 주소, 전화번호 |
| 이용 기간 설정 | 매장별 `subscription_start` / `subscription_end` 설정 |
| 점주 계정 생성 | 이메일 + 임시비번 발급, is_first_login=true 설정 |
| 매장 강제 정지 | 기간 무관 즉시 접근 차단 |

슈퍼어드민 인증은 별도 환경변수 기반 고정 계정 또는 Supabase 특수 role로 처리.

### 4-2. 점주 어드민 대시보드 (`/admin`) — 기존 유지

| 기능 | 상태 |
|------|------|
| 실시간 주문 목록 (Supabase Realtime) | ✅ 완료 |
| 주문 상태 변경 | ✅ 완료 |
| 테이블 현황 (점유/비어있음/청소중) | ✅ 완료 |
| 메뉴 CRUD (카테고리, 아이템, 옵션) | ✅ 완료 |
| 직원 계정 생성/관리 | ✅ 완료 |
| 대기 고객 관리 | ✅ 완료 |
| 일/주/월 매출 차트 | ✅ 완료 |

### 4-3. 고객 QR 메뉴 (`/m/:storeSlug/:tableId`) — 기존 유지

| 기능 | 상태 |
|------|------|
| 매장별 메뉴 조회 | ✅ 완료 |
| 테이블 자동 인식 | ✅ 완료 |
| 옵션 선택 + 장바구니 + 주문 제출 | ✅ 완료 |
| 매장 만료 시 주문 불가 처리 | ⬜ 미구현 |

### 4-4. 대기 키오스크 (`/waiting`) — 기존 유지

| 기능 | 상태 |
|------|------|
| 대기 접수 + 실시간 현황 | ✅ 완료 |

## 5. DB 스키마 변경

기존 `stores` 테이블에 컬럼 추가:

```sql
subscription_start  date          -- 이용 시작일
subscription_end    date          -- 이용 만료일
is_active           boolean       -- 강제 정지 플래그 (default true)
```

슈퍼어드민 식별:
- `store_members`에 `role = 'super_admin'` 추가 또는
- 별도 환경변수로 슈퍼어드민 이메일 화이트리스트 관리 (단순한 쪽 선택)

## 7. 남은 개발 작업

### 필수 (출시 전)

| 작업 | 설명 |
|------|------|
| **슈퍼어드민 패널** | `/superadmin` 라우트 + 매장/계정/기간 관리 UI |
| **카카오 로그인 제거** | `AuthContext`, Login 페이지에서 제거 |
| **이용 기간 접근 제어** | 만료 시 `/admin`, `/m/:storeSlug/:tableId` 차단 |
| **DB 마이그레이션** | `stores` 테이블에 `subscription_start`, `subscription_end`, `is_active` 추가 |

- `tableflow.com` 도메인 연결
