## 🟡 P1 — Phase 1 진행 중 결정

### 6. 인증 방식 + 역할/권한 설계

#### 계정 생성 플로우 (확정)
```
점주 (Owner)
  └── 카카오 소셜 로그인으로 회원가입
  └── 대시보드에서 직원 계정 생성
        └── 아이디 + 임시 비밀번호 발급
        └── 직원이 로그인 후 비밀번호 변경

직원 (Staff)
  └── 점주에게 받은 아이디/비번으로 로그인 (이메일+비번)
  └── 같은 매장 데이터만 접근 가능 (store_id 격리)
  └── 소셜 로그인 없음, 회원가입 플로우 없음
```

#### 역할별 권한 매트릭스

| 역할    | 주문 관리 | 메뉴 수정 | 직원 관리 | 매출 조회 |
|---------|----------|----------|----------|----------|
| Owner   | ✅       | ✅       | ✅       | ✅       |
| Manager | ✅       | ✅       | ❌       | ✅       |
| Staff   | ✅       | ❌       | ❌       | ❌       |

> 결제/구독 컬럼 제거 (스코프 아웃)

#### 구현 포인트 (확정)
- Supabase Auth로 카카오 소셜 로그인 처리
- `user_metadata`에 `role`, `store_id` 저장 → RLS로 자동 격리
- 직원은 Supabase Auth 이메일+비번 계정 생성 (점주가 대시보드에서 발급)
- 첫 로그인 시 임시 비밀번호 강제 변경

#### 미결 사항
- [x] Auth 라이브러리: **Supabase Auth** (1번과 일치)
- [x] 소셜 로그인: **카카오만** (점주 전용)
- [x] 직원 임시 비밀번호 전달 방식: **화면 표시 후 복사**
