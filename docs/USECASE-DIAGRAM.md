## 1. 역할 요약

| 역할 | 접근 경로 | 인증 방식 | 구독 체크 |
|------|-----------|-----------|-----------|
| **고객 (Customer)** | `/m/:storeSlug/:tableId` | 없음 (익명) | 매장 활성 여부 확인 (`checkStoreActive`) |
| **직원 (Staff)** | `/admin` → 주문 탭만 | 이메일+비번 → ProtectedRoute | 매 세션 만료/정지 체크 |
| **매니저 (Manager)** | `/admin` → 주문+메뉴+매출 탭 | 이메일+비번 → ProtectedRoute | 매 세션 만료/정지 체크 |
| **점주 (Owner)** | `/admin` → 전체 탭 | 이메일+비번 → ProtectedRoute | 매 세션 만료/정지 체크 |
| **슈퍼어드민 (Superadmin)** | `/superadmin` | 고정 계정 / Supabase 특수 role | 없음 (항상 접근 가능) |

## 4. 전체 사용자 여정

### Flow 1: 고객 QR 주문 여정

```
[고객]
  │
  ▼
QR 스캔 ──► 매장 활성 체크 ──► 비활성 ──► 에러 화면
  │                                          (UC-C25)
  │ 활성
  ▼
메뉴 화면 로드 (UC-C03)
  │
  ▼
메뉴 탐색 → 아이템 선택 → 옵션 선택 (UC-C07~C11)
  │
  ▼
장바구니 추가 (UC-C13) → 수량 변경 (UC-C14) → 장바구니 확인 (UC-C16)
  │
  ├── 빈 장바구니 ──► 주문 버튼 비활성 (UC-C17)
  │
  ▼
주문 제출 (UC-C18) ──► create_order_atomic RPC
  │
  ▼
주문 완료 (UC-C19) → 이력 조회 (UC-C20) → 상태 실시간 업데이트 (UC-C21)
```

## 5. 상태 머신

### 주문(Order) 상태

```
         created
            │
            ▼
        confirmed ◄──── (어드민 접수)
            │
            ▼
        preparing ◄──── (조리 시작)
            │
            ▼
          ready ◄──── (조리 완료)
            │
            ▼
          served ◄──── (서빙 완료)

  (any state) ──► cancelled ◄──── (취소)
```

### 대기(Waiting) 상태

```
        waiting ◄──── (대기 등록)
            │
            ▼
          called ◄──── (호출)
            │
            ▼
         seated ◄──── (착석)
            │
            ▼
        completed ◄──── (완료)

  waiting/called ──► cancelled ◄──── (취소)
  waiting/called ──► no_show ◄──── (노쇼)
```

### 테이블(Table) 상태

```
      available ◄──────────────────┐
          │                        │
          ▼                        │
       occupied ◄──── (주문 접수)   │ (청소 완료)
          │                        │
          ▼                        │
       cleaning ◄──── (퇴석)  ─────┘
```

## 6. 보안 경계

### 인증 경계

| 경로 | 접근 제어 | 미인증 시 |
|------|-----------|----------|
| `/admin` | ProtectedRoute (로그인 필수) | `/login` 리다이렉트 |
| `/superadmin` | SuperAdminRoute (슈퍼어드민 role) | `/login` 리다이렉트 |
| `/change-password` | ProtectedRoute | `/login` 리다이렉트 |
| `/m/:storeSlug/:tableId` | 없음 (공개) | — |
| `/waiting/:storeSlug` | 없음 (공개) | — |

### RLS 정책

- **멀티테넌트 격리**: 모든 데이터 접근 시 `store_id` 기반 필터링
- **크로스 스토어 차단**: 타 매장 주문/직원/매출 조회 및 수정 불가 (SEC-E01~E06)
- **역할 기반 제한**: staff는 store_settings UPDATE 불가 (SEC-E22)
- **역할 변경 차단**: manager가 자기 역할을 owner로 변경 불가 (SEC-E27)
- **고객 주문**: RLS로 INSERT만 허용

### 비밀번호 정책

- 8자 이상 + 특수문자 필수
- 첫 로그인 시 임시 비번 → 강제 변경 (`is_first_login=true`)
- 약한 비밀번호로 계정 생성 시 서버 레벨 차단 (SC-034)

### Rate Limiting

- 동일 테이블 1분 내 과도한 주문 차단 (SEC-E25)
- 대기번호 동시 호출 시 순차 번호 반환 (SC-037)
